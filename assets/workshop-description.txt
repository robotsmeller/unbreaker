Every PZ patch moves something. A file gets renamed, a global shifts scope, a
require() path changes. Mods that haven't caught up start crashing silently,
and the error log doesn't tell you which mod broke or why.

Unbreaker sits between your mods and the game's loader. When a require() call
fails because a vanilla file moved, it returns the right value instead. Real
modules always load first. Unbreaker only fires when the original fails.

134 redirects, all verified in live B42.17 sessions against a 180-mod install.
No config. No setup. Subscribe and forget it.


[h2]Is your mod covered?[/h2]

Paste your console.txt into the diagnostic tool to find out:
[url=https://robotsmeller.github.io/unbreaker/]robotsmeller.github.io/unbreaker[/url]

It tells you which require() failures Unbreaker fixes, which can't be fixed
at all, and which need work that hasn't shipped yet. Nothing is uploaded, the
log is processed in your browser.


[h2]What it does NOT fix[/h2]

Brita's Weapon Pack, Arsenal[26], True Actions. These need their authors to
rewrite parts of the mod. A redirect can't do that. Install Unbreaker and
those will still be broken.

If you're here only because of Brita or Arsenal, don't bother. It won't help.

Major library mods (damnlib, tsarslib) are on the roadmap but not stubbed yet.
Mods that depend on them may still fail until that work is done.


[h2]How it works[/h2]

Mods load by calling require() on file paths. PZ patches sometimes move those
files. When that happens the call fails and the mod crashes. Unbreaker checks
its redirect list and hands back the right value. When a mod author ships a
proper fix, the original lookup succeeds again and Unbreaker steps aside on
its own. Nothing to disable.
